# TheLastDep
Система для проведения онлайн/офлайн казино мероприятий через телеграмм бота и визуала, который транслируется игрокам.

## (Telegram-бот + клиент ведущего на PyQt6)

---

## 1. Цель проекта
Разработать программный комплекс для проведения азартной игры в формате «виртуальное казино» с участием удалённых игроков.  
Система состоит из трёх компонентов:
- **Ядро (бэкенд)** – управляет пользователями, балансами, ставками и игровой логикой.
- **Telegram-бот** – интерфейс для игроков (регистрация, просмотр баланса, приём ставок, уведомления о результатах).
- **Клиент ведущего (PyQt6)** – десктопное приложение для крупье, которое отображает ход игры на экране (для трансляции зрителям) и позволяет управлять раундами (запуск, демонстрация результата).

**Целевая аудитория:** игроки (через Telegram) и ведущий (за компьютером с демонстрационным экраном).

---

## 2. Общие требования
- Игра: **европейская рулетка** (одно зеро, 37 секторов: 0–36).
- Ставки принимаются от игроков через бота до начала вращения.
- Ведущий инициирует вращение через клиентское приложение.
- Результат генерируется бэкендом, расчёт выигрышей автоматический.
- Вся история игр и балансов сохраняется в базе данных.
- Клиент ведущего должен иметь **красивый, «казино-стиль»** интерфейс с анимацией колеса и отображением ставок игроков.
- Интерфейс должен быть адаптирован для **вывода на большой экран** (полноэкранный режим, читаемые шрифты).

---

## 3. Функциональные требования

### 3.1. Бэкенд (Flask + SQLAlchemy)
**Модели данных:**
- **User**: id (Telegram ID), username, balance (Decimal), created_at.
- **Game**: id, status (waiting, spinning, finished), result_number (0–36), created_at, finished_at.
- **Bet**: id, user_id, game_id, bet_type (enum: number, color, parity, dozen, etc.), value (int, например, номер или 0 для цвета), amount (Decimal), win_amount (Decimal, вычисляется после завершения).
- **Transaction**: id, user_id, amount, type (bet, win, deposit), timestamp.

**API эндпоинты (REST):**
- `POST /api/users` – регистрация/получение пользователя (передаётся telegram_id и username).
- `GET /api/users/<id>/balance` – текущий баланс.
- `POST /api/games` – создать новый раунд (переводит статус в waiting). Возвращает game_id.
- `GET /api/games/current` – получить текущий активный раунд (или null).
- `POST /api/bets` – сделать ставку (передаётся user_id, game_id, bet_type, value, amount). Проверяется достаточность баланса, сумма блокируется.
- `GET /api/games/<id>/bets` – список всех ставок в раунде (для клиента ведущего).
- `POST /api/games/<id>/spin` – запустить вращение (изменяет статус на spinning, генерирует result_number, затем статус finished, производит расчёт выигрышей).
- `GET /api/games/<id>/result` – получить результат (номер, цвет, список выигравших ставок с суммами).
- `GET /api/users/<id>/history` – история игр пользователя.

**Логика расчёта выигрышей:**
- Ставка на номер – выплата 35:1.
- На цвет (красное/чёрное) – 1:1.
- На чёт/нечет – 1:1.
- На дюжину (1-12, 13-24, 25-36) – 2:1.
- Другие типы (шесть линий, и т.п.) – по желанию, но для упрощения ограничимся перечисленными.

**Безопасность:** все операции с балансом должны быть транзакционны (использовать сессии SQLAlchemy с блокировками).

---

### 3.2. Telegram-бот (библиотека telebot)
**Команды:**
- `/start` – регистрация пользователя (если новый) и приветственное сообщение с правилами.
- `/balance` – показать текущий баланс.
- `/dep <сумма>` - пополнить баланс звёзд.
- `/bet <тип> <значение> <сумма>` – сделать ставку в текущем раунде. Например: `/bet number 7 100` (ставка на номер 7 суммой 100). Возвращает подтверждение или ошибку.
- `/history` – последние 10 игр с результатами и изменением баланса.
- `/help` – справка по игре.

**Интерактивные клавиатуры (для удобства):**
- Кнопки для быстрого выбора типа ставки (номер, цвет, чёт и т.д.).
- При выборе номера – клавиатура с цифрами 0–36.
- При выборе цвета – кнопки "Красное", "Чёрное".
- Ввод суммы через inline-запрос или отдельным сообщением.

**Уведомления:**
- По окончании раунда бот отправляет каждому игроку, сделавшему ставку, сообщение с результатом (выпавший номер, цвет) и суммой выигрыша (или проигрыша), а также новый баланс.
- (Опционально) рассылка всем игрокам в группе или личные сообщения.

**Ограничения:**
- Ставки принимаются только в статусе `waiting`.
- Минимальная и максимальная ставка (задаются конфигом, например, 1 и 1000).

---

### 3.3. Клиент ведущего (PyQt6 + requests)
**Функции:**
- Авторизация ведущего (локально, без пароля – просто запуск приложения).
- Подключение к бэкенду (указание URL, например, `http://127.0.0.1:5000`).
- Отображение текущего раунда:
  - Статус (ожидание ставок, вращение, завершён).
  - Список всех ставок, сгруппированных по типу (например, все ставки на номер 7, на красное и т.д.) с указанием общего количества поставленных сумм.
  - Крупные цифры для демонстрации на экране.
- Кнопка **«Spin»** (активна только в статусе `waiting`). При нажатии отправляет `POST /api/games/<id>/spin`, после чего бэкенд генерирует результат.
- Анимация вращения колеса рулетки (использовать QGraphicsScene + QPropertyAnimation или QTimer для вращения картинки). Длительность ~5–8 секунд.
- По окончании анимации – отображение результата:
  - Крупный номер в центре.
  - Цвет сектора.
  - Подсветка выигравших ставок (например, зелёным цветом) и отображение сумм выигрыша по каждому игроку (можно в виде бегущей строки).
- Возможность запустить новый раунд (кнопка «New Round»), которая создаёт новую игру через `POST /api/games` и переводит интерфейс в состояние ожидания ставок.

**Интерфейсные требования:**
- Тёмный фон, золотые/красные акценты.
- Полноэкранный режим (F11 или кнопка).
- Адаптивное масштабирование (чтобы на большом экране всё было читаемо).

## 4. Нефункциональные требования
- Бэкенд должен обрабатывать до 50 одновременных игроков без значительных задержек.
- Время ответа API – не более 1 секунды.
- Бот должен работать стабильно, сообщения доставляться в течение нескольких секунд.
- Клиент ведущего должен запускаться на Windows 10/11 (PyQt6 кросс-платформенный, но для простоты настройка под Windows).
- Вся конфигурация (порт бэкенда, токен бота, настройки ставок) выносится в файл `.env` или `config.py`.

## 5. Ход игры (пошаговый сценарий)

1. **Ведущий** запускает клиент PyQt6, подключается к бэкенду.  
   Приложение показывает экран с сообщением «Ожидание начала раунда». Ведущий нажимает **«New Round»** – создаётся новая игра со статусом `waiting`.

2. **Игроки** через Telegram-бота видят, что начался новый раунд (можно сделать оповещение в группу или через рассылку). Они делают ставки, используя команды или кнопки.  
   Бэкенд проверяет достаточность баланса, резервирует сумму и записывает ставку.

3. **Ведущий** на экране клиента видит обновляющийся список ставок (каждую секунду опрашивает API или использует WebSocket – для упрощения используем периодический опрос). Когда ставки приняты, ведущий нажимает **«Spin»**.

4. Бэкенд получает запрос на вращение, переводит игру в статус `spinning`, генерирует случайное число (0–36) и вычисляет цвет (красное/чёрное/зелёное для 0).  
   Затем производит расчёт выигрышей для всех ставок, обновляет балансы игроков, фиксирует транзакции, сохраняет результат. Статус игры становится `finished`.

5. Клиент ведущего опрашивает результат (`GET /api/games/<id>/result`), запускает анимацию вращения колеса.  
   По окончании анимации показывает выпавшее число и цвет, а также выделяет выигравшие ставки (например, подсвечивает сектора на колесе и выводит список победителей с суммами).

6. Бэкенд (или отдельный поток) инициирует отправку уведомлений игрокам через Telegram-бота. Каждый игрок получает сообщение с результатом и новым балансом.

7. Ведущий может начать новый раунд, нажав **«New Round»** – процесс повторяется.

## 6. Технический стек и требования к реализации

| Компонент         | Технологии                                                                 |
|-------------------|----------------------------------------------------------------------------|
| Бэкенд            | Python 3.10+, Flask, SQLAlchemy, SQLite                                    |
| Telegram-бот      | pyTelegramBotAPI (telebot), requests для вызовов API бэкенда               |
| Клиент ведущего   | Python 3.10+, PyQt6, requests, (опционально QtCharts для графиков)         |
| Дополнительно     | python-dotenv для конфигурации, логирование через logging                  |

**Архитектура взаимодействия:**  
- Бэкенд – единственная точка доступа к данным. Бот и клиент ведущего общаются с ним по HTTP.  
- Для уведомлений бот использует long-polling (или webhook, но для простоты – polling).  
- Клиент ведущего опрашивает бэкенд каждые 2 секунды для обновления списка ставок (можно использовать SSE или WebSocket, но для упрощения оставим polling).
